Area de GIT

Working Directory

Es el área de trabajo, donde rara vez vamos a estar utilizaando la línea de comandos, salvo por el inicio del proyecto o a lo sumo, la elimilación del mismo. Se refiere a la carpeta donde iniciamos el repositorio y basicamente donde vamos a crear, modificar y eliminar los archivos que forman parte de nuestro proyecto.

  • dir || lsmuestra el contenido de la carpeta actual
  • mkdir <name>crea una carpeta con el nombre indicado
  • cd <nameabre la carpeta con el nombre indicado
  • git initinicializa un repositorio en Git
  • rm -rf .giteliminar el repositorio de manera forzada

Staging Area(INDEX)

El area de control de cambio corresponde a un archivo que se encuentra en la carpeta .git yse utiliza para realizar las carpetas de pantallas del código (Snapshot) de los diferentes archivos del proyecto. Aquí se realiza el siguimiento de los mismo, para realizar las confirmación y poder acceder a los cambios dentro de la historia de nuestro proyecto. Es importante tener en cuenta que todo archivo que no sea agregado al INDEX, no se podra recuperar en cado de ser eliminado.

  • git status
    • muestra el estado de los archivos con respecto a la última confirmación, en caso que exista.
    • untracked: si al archivo no se le está realizando seguimiento
    • staged: el archivo o los cambios fueron agregado, pero, aun no se han confirmado
    • modified: el archivo difiere con respecto a lo almacenado en la última confirmación
    • commited: los cambios fueron almacenados en una confirmación
  • git add <file>agrega al INDEX el/los archivos seleccionados
  • git re --filequita del INDEX el archivo seleccionado
  • git resstore --stage <file>remueve los cambios de los archivos agregados al INDEX

  • git statusmuestra el estado de los archivos
